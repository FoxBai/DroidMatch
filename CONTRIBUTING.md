# Contributing

DroidMatch is in the M1 harness phase. Keep changes small, verifiable, and aligned with the current placeholder boundaries in `README.md`, `mac/README.md`, and `android/README.md`.

Before opening or merging a change, run the relevant gates:

```text
bash tools/check-env.sh --all
bash tools/check-m0.sh
bash tools/check-proto.sh
bash tools/check-m1-run-logs.sh
bash tools/check-m1-skeleton.sh
```

`tools/check-env.sh` is a preflight check for host prerequisites. It reports
missing `protoc`, Swift Testing, Java, Android SDK, and Gradle problems in both
English and Chinese before the longer build logs start.

`tools/check-env.sh` 是本机依赖 preflight，会在长构建日志开始前用中英文报告
`protoc`、Swift Testing、Java、Android SDK 和 Gradle 的缺失问题。

Mac-only changes should also run:

```text
bash tools/run-swift-tests.sh
```

Android APK changes should run through the checked-in Gradle wrapper:

```text
cd android
./gradlew --no-daemon :app:testDebugUnitTest :app:assembleDebug :app:lintDebug
```

If dependency downloads are flaky but the artifacts are already in the local Gradle cache, rerun the M1 gate with `DROIDMATCH_GRADLE_OFFLINE=1 bash tools/check-m1-skeleton.sh`.

Android Java sources depend on protobuf classes generated by Gradle, so `check-m1-skeleton.sh` treats the Gradle build as the authoritative Android compile check.

Regenerating Swift protobuf files requires `protoc` on `PATH` in addition to SwiftPM resolving `swift-protobuf`; on macOS, install it with `brew install protobuf`.

GitHub Actions gates are documented in `docs/ci-cd.md`. CI covers spec,
protobuf, Swift harness, Android skeleton, and redacted run-log checks. Physical
device smoke tests are not run in CI; record those with
`tools/run-m1-device-smoke.sh` and commit redacted logs only when they prove a
new M1 matrix case.

GitHub Actions gate 见 `docs/ci-cd.md`。CI 覆盖规格、protobuf、Swift harness、Android
skeleton 和脱敏 run-log 检查。物理设备 smoke 不放进 CI；请用
`tools/run-m1-device-smoke.sh` 记录，并只在证明新的 M1 矩阵场景时提交脱敏日志。

When changing protocol, security, transport, Android service behavior, or Mac harness behavior, update the matching docs in the same change. Do not reuse HandShaker code, assets, binaries, signing material, branding, or copied UI implementation.

When adding durable comments or docs for shared workflows, prefer paired English
and Chinese wording so local and CI failures remain easy to hand off.

为共享流程新增长期注释或文档时，优先同时写英文和中文，方便本地与 CI 失败时交接。
