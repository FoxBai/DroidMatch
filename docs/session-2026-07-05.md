## Remaining M1 Blockers

### High Priority (Require Physical Devices)

1. **Handshake stability test:**
   - Need 20 attempts with ≥19 passes
   - Can run on existing NIO N2301 device
   - Command ready in testing guide

2. **Throughput validation:**
   - Need 100MB download/upload with throughput assertions
   - Requires `--chunk-size-bytes 1048576 --min-download-mib-per-second 20`
   - Can run on existing NIO N2301 device
   - Command ready in testing guide

3. **Device matrix expansion:**
   - Need Slot A device (API 26-29)
   - Need Slot C device (API 33-35)
   - Both can reuse test scenarios from quick-test-scenarios.sh

### Medium Priority (Implementation)

4. **Multi-stream scheduling:**
   - Protocol supports `stream_id` multiplexing
   - Need to implement scheduler for 2 concurrent transfers
   - Estimated effort: ~2-3 days

5. **Full recovery queue:**
   - Currently only one automatic retry
   - Need exponential backoff and multiple attempts
   - Estimated effort: ~1-2 days
